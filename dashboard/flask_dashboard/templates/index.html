Required metrics for section 3.1.5

CategoryMetrics
Functional testingController API health, VIP ARP response, backend JSON reply, health toggle effect, GA recompute success
PerformanceHTTP req/s, p50 latency, p95 latency, error rate, SLA compliance %, active flows
ScalabilityThroughput under concurrency 10 / 20 / 50, backend distribution balance, bandwidth utilization
Resource usagePer-backend CPU %, memory %, bandwidth %, active connections, aggregate throughput Mbps
Fault toleranceService continuity after backend failure, traffic shift time, reduced but stable throughput, error spike duration
Prometheus is optional. Even without it, the controller already exposes flow count and bandwidth-derived metrics using OpenFlow port statistics. Prometheus adds CPU and memory utilization per backend when you map exporter instances in config.controller.yaml.
